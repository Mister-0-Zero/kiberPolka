# Data Fusion Contest 2026 — Киберполка
## Базовый README проекта

## Цель проекта

Решить задачу **multi-label классификации** для 41 финансового продукта клиентов банка на основе анонимизированных табличных данных.

Каждому клиенту необходимо предсказать вероятность открытия каждого из продуктов.

Метрика соревнования:

**Macro ROC-AUC**

То есть среднее значение ROC-AUC по всем 41 таргету.

Важно:

- каждый таргет имеет одинаковый вес
- редкие классы влияют на итоговую метрику так же, как частые
- стабильность по всем таргетам критична

## Структура данных

### Train

### Файлы:

```text
train_main_features.parquet
train_extra_features.parquet
train_target.parquet
```

### Описание:

```text
train_main_features.parquet — основной набор признаков (~200)
train_extra_features.parquet — дополнительный набор признаков (>2000)
train_target.parquet — целевые переменные (41 таргет)
```

**Ключ объединения:**

customer_id

### Размер:

~750 000 объектов
~2200+ признаков
41 таргет
Test

### Файлы:

```text
test_main_features.parquet
test_extra_features.parquet
sample_submit.parquet
```

### Описание:

данные аналогичны train
отсутствуют target
необходимо сформировать предсказания

### Размер:

~250 000 объектов
Формат submission

### Файл должен быть строго в формате:

.parquet

### Структура:

customer_id
predict_1
predict_2
...
predict_41

### Требования:

порядок столбцов должен совпадать с sample_submit.parquet
названия столбцов должны совпадать
количество строк должно совпадать с test

Любое отклонение приведет к ошибке отправки.

### Метрика

Используется:

from sklearn.metrics import roc_auc_score

roc_auc_score(
    y_true,
    y_pred,
    average="macro"
)

### Расчет:

Macro AUC = mean(AUC_1 ... AUC_41)

### Следствия:

необходимо контролировать качество по каждому таргету
один слабый таргет снижает общий результат
важно логировать AUC по каждому таргету отдельно
